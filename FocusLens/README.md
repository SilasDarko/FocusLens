# FocusLens

**On-Device Accessibility Study Assistant**

> A portfolio iOS application demonstrating privacy-preserving, on-device machine learning, MVVM architecture, accessibility-aware SwiftUI design, and local-only data persistence.

---

## Why This Project

Most productivity apps collect behavioral data and run inference in the cloud. FocusLens takes the opposite approach: a student's focus patterns, interruption frequency, and stress self-ratings never leave their device. Every prediction is generated by a Core ML model running locally — even without a network connection.

This project was built to demonstrate:

- **On-device ML integration** using Core ML and a Create ML training workflow
- **Privacy-first architecture** — no network calls, no analytics, no cloud dependency
- **Accessibility-aware UI** using SwiftUI Dynamic Type, VoiceOver labels, reduced motion, and semantic button design
- **Clean MVVM structure** with testable view models and a clear service boundary
- **Honest prototype scope** — sample/labeled training data, documented limitations, and fallback behavior when the ML model is not yet available

## Features

### 1. Privacy-First Onboarding
A four-page onboarding flow explains the privacy model before any data is collected: on-device-only predictions, local storage, no backend, and the ability to delete all history.

### 2. Session Setup
Students configure each study session with:
- Subject and goal
- Planned duration
- Environment type (Quiet, Library, Dorm, Shared Space, Outdoors, Noisy)
- Energy and distraction levels
- Accessibility preference notes

### 3. Active Session
A live timer with a circular progress ring, checkpoint logging, and an "End Session" flow. Checkpoints capture:
- Self-rated focus (1–5)
- Stress level (1–5)
- Perceived difficulty (1–5)
- Interruption count
- App/task switching

### 4. On-Device Focus Prediction
After a session ends, `FocusPredictionService` generates a prediction using the bundled Core ML tabular classifier. If the model is unavailable (e.g. before training), a deterministic heuristic fallback ensures the app is always functional.

**Labels:** `Deep Focus`, `Mixed Focus`, `Distracted`, `Recovery Needed`

### 5. Prediction Result Screen
Displays:
- Predicted category and confidence percentage
- Contributing factor explanations (e.g. "High interruption count logged across checkpoints")
- A reflection prompt — phrased as a study suggestion, not medical advice
- Session summary stats

### 6. History
A chronological list of completed sessions. Each row shows subject, predicted category, confidence, duration, and the key friction point. Swipe to delete individual sessions or remove all history from Settings.

### 7. Insights
Local calculations over past session data:
- Most common focus category
- Average interruptions per session
- Best-performing study environment
- Focus score trend chart (Swift Charts)

No second ML model. All insights use deterministic arithmetic.

### 8. Accessibility
- Full **Dynamic Type** support — no fixed font sizes
- **VoiceOver labels** on all interactive and informational elements
- **Reduced motion** preference respected via `AccessibilitySettingsService`
- **High contrast** toggle in Settings
- RatingRow supports `accessibilityAdjustableAction` for VoiceOver swipe-to-adjust
- Form fields use `.accessibilityHint` to explain required input
- Semantic button labels (never "tap here")

### Key Design Decisions

| Decision | Rationale |
|---|---|
| JSON over SwiftData | Avoids Xcode version constraints; portable, debuggable, easy to reset |
| Heuristic fallback | App is functional before Create ML training is complete |
| `SessionFeatures` struct | Decouples raw session data from the ML input format; independently testable |
| `@MainActor` ViewModels | Ensures UI state updates happen on the main thread without manual dispatch |
| No backend / no auth | Privacy guarantee is architectural, not just a setting |

## On-Device ML Explanation

FocusLens uses a **tabular classifier** trained in Create ML on 10 numeric features extracted from a completed study session. The model file (`FocusLensModel.mlmodel`) is bundled in the app at build time and executed by Core ML.

**The model never runs on a server.** Core ML handles scheduling, device capability detection, and memory management automatically.

**Features used:**
- Planned duration, checkpoint count
- Average self-rated focus, stress, difficulty
- Total interruptions, app switch count
- Energy level, pre-session distraction level, environment type

**Training data:** 50 hand-labeled synthetic sessions (see `ML/training_data.csv`). This is demonstration data. Real deployment would require real labeled sessions (200+ per class recommended).

## Privacy Design

| Category | Approach |
|---|---|
| Session data | Stored as local JSON in the app's Documents directory |
| Predictions | Computed on-device; stored alongside session data |
| Analytics | None |
| Crash reporting | None |
| Network calls | None |
| Data deletion | Single session or all history, user-initiated from History / Settings |

The privacy model is enforced architecturally: `LocalStorageService` reads/writes only to the local filesystem. `FocusPredictionService` loads the model from the app bundle at init and calls `MLModel.prediction()` synchronously. There is no `URLSession` or network stack in the codebase.

---

## Accessibility Considerations

- All text uses system fonts that scale with Dynamic Type
- Tap targets are minimum 44×44pt
- VoiceOver labels describe every interactive element's purpose, not just its appearance
- Reduced motion preference is respected via `AccessibilitySettingsService.preferredAnimation`
- RatingRow implements `accessibilityAdjustableAction` so VoiceOver users can swipe up/down to change ratings
- Color is never the sole differentiator of information (category badges use text labels too)
- The app is designed for 4.7" screens up; no horizontal-only layouts

---

## Limitations

- **Sample data is synthetic.** The 50-row CSV is hand-labeled for demonstration. A real deployment would need genuine labeled sessions.
- **No passive sensing.** All inputs are self-reported. The app does not access the accelerometer, microphone, camera, or screen time API.
- **Single-session predictions only.** The model predicts from session-level aggregates; it does not model within-session temporal sequences.
- **Not a clinical tool.** Focus predictions are study-pattern reflections, not psychological or medical assessments. The app explicitly disclaims this in onboarding and on the result screen.
- **Confidence is relative.** Confidence scores reflect how closely a session matched training examples — not absolute certainty.

## How to Run

### Requirements
- Xcode 15.3+
- iOS 17+ deployment target
- Swift 5.9+

### Steps

1. Clone the repository
2. Open `FocusLens.xcodeproj` in Xcode
3. Select a Simulator or connected device
4. Build and run (`⌘R`)

The app runs fully without the trained ML model (heuristic fallback is active). To enable Core ML predictions, follow the training steps in `FocusLens/ML/README_ML.md`.

### Running Tests

```bash
xcodebuild test -scheme FocusLens -destination 'platform=iOS Simulator,name=iPhone 15'
```

Or use `⌘U` in Xcode.

## Future Improvements

- **Temporal modeling:** Sequence models (e.g. LSTM via Core ML) that predict within-session focus degradation rather than only end-of-session labels
- **Passive signal integration (opt-in):** Accelerometer patterns or Screen Time API data, with explicit user consent and on-device-only processing
- **HealthKit integration (opt-in):** Sleep duration as a pre-session feature, with privacy controls matching HealthKit standards
- **Federated learning exploration:** On-device model personalization without uploading raw data
- **Widget:** A session quick-start or streak widget using WidgetKit
- **Shortcut support:** Start session via Siri Shortcuts
- **Export:** Allow users to export their own session data as JSON or CSV
